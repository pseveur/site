Bienvenue à Formation Git

Introduction

Ce document a pour but de présenter les exercices réalisés lors de la formation Git pour développeur.

Dans ce document, Git est toujours utilisé depuis Eclipse. Il ne s’agit donc pas ici d’utiliser Git en lignes de commande ou avec d’autres clients. Il est donc indispensable que l’utilisateur de ce guide connaisse Eclipse et sache un minimum développer avec cet outil.

Ce document peut être utilisé lors de la formation mais également de façon individuelle par toute personne souhaitant avoir une première approche Git. Il est suffisamment illustré pour pouvoir appréhender seul les principaux concepts et surtout pratiquer.

Pour la formation, se connecter sur le poste avec son propre matricule (a ou x) et non celui de la salle de formation.

gitflow-hotfix.puml

@startuml
participant develop
participant master
master -> master: tag 1.2.0
create participant "hotfix/1.2.1" as hotfix
master -> hotfix: branch from master
activate hotfix
hotfix -> hotfix: commit
hotfix -> hotfix: commit
note over hotfix #D3D3D3: 1.2.1-beta.1+2
hotfix -> hotfix: tag 1.2.1-beta.1
note over hotfix #D3D3D3: 1.2.1-beta.1

  Hotfix branches are deleted once merged
end note

1.1 Configurer https

Il convient de désactiver le paramètre de connexion sslverify.

  1. Ouvrir les préférences Window > Preferences > Team > Git > Configuration.
  2. Puis sous l’onglet User Settings, ajouter une entrée http avec sslverify affecté à false.
    Utiliser le bouton Add Entry... Attention le nom de la clé à fournir est http.sslverify

Le résultat obtenu est le suivant :

Figure 2- Configuration https

1.2 Configurer l’utilisateur

Il convient de renseigner le nom et l’adresse mail de l’utilisateur.

  1. Ouvrir les préférences Window > Preferences > Team > Git > Configuration.
  2. Puis sous l’onglet User Settings, ajouter une entrée user avec email affecté à votre adresse mail et avec name affecté avec votre matricule, prénom et nom.
    Utiliser le bouton Add Entry... Attention les noms des clés à fournir sont user.email et user.name

Le résultat obtenu est le suivant :

Figure 3- Configuration Utilisateur

1.3 Configurer la connexion réseau

Il convient de modifier les paramètres de connectivité réseau.

  1. Ouvrir les préférences Window > Preferences > General > Network Connections.
  2. Sélectionner comme Active Provider l’option Native.
  3. S’assurer que le « Schema » HTTPS utilise bien le proxy et que le suffixe « .socgen » ne passe pas par le proxy.
